# OpenClaw CN - 中国商用个人助手

## 愿景

打造面向中国市场的**商用级个人 AI 助手**，帮助用户通过熟悉的消息渠道高效完成日常工作。

## 技术基础

基于 [OpenClaw](https://github.com/openclaw/openclaw) 架构：
- **技术栈**: TypeScript/Node.js (ESM), pnpm
- **运行时**: Node 22+
- **架构**: Gateway (WebSocket 控制平面) + Agent 运行时
- **部署平台**: macOS / Linux / Windows

## 待讨论

### AI 模型策略

商用需要考虑稳定性和合规性：

| 方案 | 优点 | 缺点 |
|------|------|------|
| **国产模型** (通义/文心/智谱/DeepSeek) | 合规、低延迟 | 能力差异 |
| **API代理** (Claude/GPT) | 能力强 | 稳定性风险、合规风险 |
| **私有部署** (开源模型) | 完全可控 | 成本高、运维复杂 |
| **多模型混合** | 灵活 | 复杂度高 |

**问题**: 你倾向哪种方案？

### 商业模式

- **SaaS服务**: 托管部署，按使用量收费
- **私有部署**: 一次性授权 + 维护费
- **混合模式**: 基础功能免费，高级功能收费
